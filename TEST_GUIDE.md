# Vivo 安卓手机测试指南

## 📱 测试前准备

### 1. 启用 Vivo 手机的开发者选项

#### 步骤：
1. 打开 **设置** → **关于手机**
2. 连续点击 **版本号** 7 次（直到提示"您已处于开发者模式"）
3. 返回 **设置** → **系统管理** → **开发者选项**

### 2. 开启 USB 调试

1. 进入 **开发者选项**
2. 找到并开启 **USB 调试**
3. 弹出确认框时，点击 **确定**

### 3. 连接手机到电脑

1. 使用 USB 数据线连接 Vivo 手机到电脑
2. 手机上弹出"允许 USB 调试"时，选择 **允许**
3. 选择 **传输文件** 模式

## 🔍 验证连接

### 在电脑上检查手机是否连接：

```bash
# 进入项目目录
cd C:\Users\admin\Desktop\xmwj\dianhua

# 检查设备连接
flutter devices
```

**预期输出：**
```
1 connected device:
V1824A (mobile) • XXXXXXXX • android-arm64 • Android 13 (API 33)
```

如果看不到设备：
- 检查 USB 线是否连接好
- 重新插拔 USB 线
- 重启开发者选项
- 安装 Vivo USB 驱动（通常 Windows 会自动安装）

## 🚀 运行应用测试

### 方法一：直接运行（推荐新手）

```bash
# 在项目目录下运行
flutter run
```

**运行过程：**
1. 首次运行会下载 Gradle 和依赖（可能需要几分钟）
2. 自动编译 APK
3. 安装到手机
4. 自动启动应用

**预期时间：** 首次约 3-5 分钟，后续约 30 秒

### 方法二：先构建再安装

```bash
# 1. 构建 APK
flutter build apk --debug

# 2. 安装到手机
flutter install
```

APK 位置：`build\app\outputs\flutter-apk\app-debug.apk`

## 🐛 常见问题排查

### 问题 1：设备未识别

**症状：** `flutter devices` 看不到手机

**解决方案：**
```bash
# 1. 检查 ADB
adb devices

# 2. 重启 ADB
adb kill-server
adb start-server

# 3. 重新连接 USB
# 拔掉 USB 线，重新插入
```

### 问题 2：权限被拒绝

**症状：** 相机无法打开

**解决方案：**
1. 手机设置 → 应用管理 → call_identifier
2. 权限管理 → 授予相机权限
3. 重新启动应用

### 问题 3：安装失败

**症状：** 安装时提示解析包错误

**解决方案：**
```bash
# 清理构建缓存
flutter clean

# 重新获取依赖
flutter pub get

# 重新构建
flutter build apk --debug

# 重新安装
flutter install
```

### 问题 4：应用闪退

**症状：** 应用启动后立即关闭

**解决方案：**
```bash
# 查看日志
flutter logs

# 或者使用 adb
adb logcat | findstr "call_identifier"
```
